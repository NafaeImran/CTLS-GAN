# CTLS-GAN: Inducing Class Targeted Label Skew using GANs to Steer Global Model

CTLS-GAN is a novel federated learning attack framework that uses Generative Adversarial Networks to produce high quality synthetic data for specific classes in order to artificially induce a label skew and degrade global model performance. The attack operates in Non-IID federated environments, where it exploits natural data heterogeneity to remain indistinguishable from a benign skewed client.

## Project Overview

Federated Learning (FL) enables collaborative model training across decentralized devices while preserving data privacy. FL's aggregation method is inherently vulnerable to data poisoning attacks, since it assumes the legitimacy of all participating clients. Prior work demonstrates GAN-based data poisoning that degrades global model performance by introducing noise across all classes indiscriminately. However, it does not explore introducing label skew by generating synthetic data to bias the global model toward a specific direction.

CTLS-GAN addresses this gap. Multiple malicious clients pose as valid clients and replace their local training data with large amounts of synthetic images generated by GANs, biasing their local models toward specific classes. This bias then propagates to the global model through aggregation. The attack is stealthy because it requires no explicit modification to gradients and instead mimics naturally occurring label skew in Non-IID environments.

We demonstrate the attack against several aggregation strategies and defense mechanisms:

1. **FedAvg**: Standard federated averaging, serving as the baseline
2. **FedCC**: Byzantine-robust defense using CKA similarity scoring to filter malicious clients
3. **MOON**: Contrastive learning-based aggregation designed to deal with Non-IID data
4. **SignGuard**: Gradient statistics-based defense using mean shift clustering

CTLS-GAN successfully bypasses all four strategies, demonstrating a clear gap in existing defense mechanisms.

### Running the Code

1. Clone this repository:
```bash
git clone https://github.com/NafaeImran/CTLS-GAN.git
cd CTLS-GAN
```

2. Install dependencies:
```bash
pip install torch torchvision numpy matplotlib pandas jupyter
```

3. To reproduce a specific experiment, open the corresponding notebook under `tests/`. For example, to run the FedCC attack with the default configuration (beta = 0.12, seed = 67, 1-class):
```bash
jupyter notebook tests/dl-proj-fedcc-beta0.12-seed67-class1.ipynb
```

4. To reproduce the ablation study across beta values:
```bash
jupyter notebook tests/dl-proj-betatests.ipynb
```

5. To inspect bypass rate (stealthiness analysis):
   Copy the logs into a file in the logs folder, and then run this notebook on the newly created log file.
```bash
jupyter notebook logs/StealthAnalysis.ipynb
```

## Key Innovations

### Attack Formulation

CTLS-GAN trains a GANs model using the local data found on a taken over client's to generate high quality synthetic images for targeted classes. For the purposes of this paper we used NVIDIA's StyleGAN2-ADA pretrained on CIFAR-10. Malicious clients train their local models on a mixture of real and synthetic data, where the synthetic data is heavily skewed toward the targeted class. The resulting gradients closely resemble those of a legitimately skewed benign client in a Non-IID environment, making the attack inherently stealthy.

The attack is formulated over one global model and n = 10 clients, of which m = 3 are adversaries. Both benign and malicious clients use the same CNN architecture (CifarCNN):

| Input | Image | 3x32x32 | RGB Input |

### Attack Growth Rate (Beta)

The hyperparameter beta controls the rate at which the proportion of synthetic data used for training malicious clients increases per communication round. A lower beta means a slower ramp-up, preserving stealthiness. A higher beta means a more aggressive attack at the cost of potential detection.

| Growth Rate (beta) | Adversarial Profile |
|-------------------|---------------------|
| 0.005 | Persistence: Long-term stealth, minimal per-round gradient deviation |
| 0.020 | Conservative: Gradual poisoning to stay within clustering thresholds |
| 0.050 | Moderate: Balanced trade-off between detection risk and model steering |
| 0.120 | Aggressive (Default): Rapid injection of GAN-generated label skew |
| 1.000 | Maximal: Immediate saturation of local training with synthetic data |

To prevent detection via anomalous training time, the total data volume (synthetic + real) on each malicious client is kept equal to its original local dataset size.

### Targeted Class Count (N-Top)

The ntop parameter controls how many of the highest-data classes are targeted. When ntop = 1, the attack concentrates on a single class and produces the strongest per-class bias. When ntop >= 2, the attack randomly cycles among the top classes each round, distributing the skew more naturally, which reduces per-class anomaly but also dilutes misclassification pressure on any single target.

## Experimental Results

All experiments use CIFAR-10 (50K training images, 10K test images) distributed across 10 clients via Dirichlet distribution (alpha = 0.5). The default attack uses beta = 0.12.

### Robustness Evaluation
<img width="4369" height="8192" alt="ExperimentSetup" src="https://github.com/user-attachments/assets/f28b3b7a-956c-43be-aeba-09eeda800c06" />


| Strategy | Non-IID (1-Class) | Attack (1-Class) | Non-IID (2-Class) | Attack (2-Class) |
|----------|------------------|-----------------|------------------|-----------------|
| FedAvg | 72.99% | 68.73% | 72.41% | 70.20% |
| FedCC | 70.52% | 65.68% | 72.15% | 67.19% |
| MOON | 64.44% | 53.53% | 63.43% | 59.63% |
| SignGuard | 70.38% | 65.08% | 66.25% | 67.28% |

Key observations:

- **FedAvg**: Performance degraded by 2.5% in the 1-class scenario and 4.2% in the 2-class scenario. Confusion matrices show clear bias toward the targeted class (cat).
- **FedCC**: The attack bypassed FedCC's CKA filter up to 92% of the time for Client 1. Overall accuracy dropped by roughly 5% in the 1-class run and 4.8% in the 2-class run.
- **MOON**: The most severely affected strategy, suffering an 11% drop under the 1-class attack. MOON's contrastive objective reinforces biased representations rather than correcting them, making it particularly vulnerable to CTLS-GAN.
- **SignGuard**: Partial resistance. The 1-class attack caused a 5% accuracy reduction, but the 2-class attack could not degrade accuracy. Splitting the attack objective across two classes disperses updates enough that SignGuard's mean shift clustering neutralizes them.

Stealthiness analysis (L2 divergence and cosine similarity) confirms that malicious clients [1, 2, 5] are indistinguishable from benign ones throughout training. There is no concrete way to identify a malicious client using divergence or similarity metrics alone.

### Ablation Study

Ablation runs were performed on a subset of CIFAR-10 (20K training, 10K test images).

| Scenario (beta) | FedAvg (%) | FedCC (%) |
|----------------|-----------|----------|
| Baseline (Non-IID) | 63.77 | 62.81 |
| Attack (beta = 0.005) | 62.43 | 60.84 |
| Attack (beta = 0.02) | 59.09 | 56.93 |
| Attack (beta = 0.05) | 57.22 | 50.54 |
| Attack (beta = 0.12) | 57.28 | 57.91 |
| Attack (beta = 1.0) | 55.00 | 55.32 |

Notably, FedCC drops to 50.5% at beta = 0.05 but partially recovers at beta = 0.12, suggesting that its clustering mechanism partially adapts to more aggressive injection rates.
